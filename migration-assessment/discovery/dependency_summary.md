# Dependency Summary - Spring PetClinic (Module 1.3)

Classification of every discovered dependency for the PetClinic workload, derived from
`workload_inventory.json` (generated by `k8s_inventory_crawler.py`) plus observed runtime
behavior. Each dependency is tagged with a migration treatment used later in the wave plan.

## Workloads

| Workload | Kind | Runtime | Port | Notes |
|---|---|---|---|---|
| `petclinic` | Deployment | Java 21 / Spring Boot 4 | 8080 | Stateless monolith, non-root container |
| `postgres` | StatefulSet | PostgreSQL 18 | 5432 | Stateful, backed by a PVC |

## Dependency classification

| Type | Dependency | Direction | Detail | Migration treatment |
|---|---|---|---|---|
| **Ingress** | External HTTP(S) clients -> app | inbound | Via ingress controller, host `petclinic.local`, TLS at edge | NGINX ingress on AKS + DNS/TLS cutover |
| **Service-to-service** | `petclinic` -> `kube-dns` | egress | DNS resolution (53) | Allowed explicitly in egress policy |
| **Database** | `petclinic` -> `postgres:5432` | egress | JDBC `jdbc:postgresql://postgres:5432/petclinic` | **Replatform** to Azure DB for PostgreSQL Flexible Server |
| **Message queue** | _none_ | - | PetClinic has no MQ dependency | n/a (would apply to AKS Store demo) |
| **Filesystem** | `postgres` -> PVC | mount | DB data volume (`local-path` on-prem) | **Managed CSI** (Azure Disk) or eliminated when DB is managed |
| **External API** | `petclinic` -> partner API (modeled) | egress | HTTPS 443 to an approved FQDN | Cilium FQDN allowlist (Module 5) |
| **Secrets** | `petclinic`/`postgres` -> `demo-db` Secret | mount/env | DB credentials | **Key Vault + Secret Store CSI** + workload identity on AKS |
| **Config** | `petclinic` -> `petclinic-config` ConfigMap | env | Spring profile, JDBC URL, actuator flags | Per-environment Helm values overlays |

## Dependency graph

    internet client --HTTPS 443--> ingress-nginx --HTTP 8080--> petclinic
    petclinic --DNS 53--> kube-dns
    petclinic --TCP 5432--> postgres            (postgres mounts its PVC and the demo-db Secret)
    petclinic --HTTPS 443--> partner API        (approved FQDN only)
    petclinic --mount--> demo-db Secret
    unauthorized namespace --X blocked--> petclinic

## Source -> target dependency changes

| On-prem (Rancher/K3s) | AKS target |
|---|---|
| In-cluster `postgres` StatefulSet + PVC | Azure DB for PostgreSQL Flexible Server (private endpoint) |
| Plain Kubernetes `Secret` | Azure Key Vault + Secret Store CSI (workload identity) |
| `local-path` storage | Azure managed CSI (Disk/Files) - only if DB stays in-cluster |
| Traefik/NodePort exposure | ingress-nginx + Azure LB + DNS/TLS cutover |
| No network policy | Default-deny NetworkPolicy + CiliumNetworkPolicy egress allowlist |
| Local image (`petclinic:onprem`) | Multi-arch image in ACR, pulled via AcrPull identity |
